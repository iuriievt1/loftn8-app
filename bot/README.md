# Farshiki Telegram Bot

## Как создать и запустить

1. Открой в Telegram `@BotFather`.
2. Напиши `/newbot`, задай имя и username бота.
3. Скопируй токен, который даст BotFather.
4. Напиши своему боту любое сообщение, потом открой в браузере:

```text
https://api.telegram.org/botТОКЕН/getUpdates
```

5. Найди `chat.id` своего аккаунта. Это будет `ADMIN_CHAT_ID`.
6. Создай файл `bot/.env` по примеру `bot/.env.example`.
7. Укажи `BOT_TOKEN`, `ADMIN_CHAT_ID`, данные получателя СБП и SMTP, если нужны письма.
8. Установи зависимости:

```bash
npm install
```

9. Запусти сайт, API заказов и бота одной командой:

```bash
npm start
```

Если нужно запустить только бота отдельно:

```bash
npm run bot
```

## Как работает сценарий

Покупатель оформляет заказ на сайте. Сайт отправляет заказ в `/api/orders`, сохраняет номер заказа и переводит покупателя в Telegram:

```text
https://t.me/farshikistore?start=order_FS-1234567
```

Бот находит заказ, показывает состав, сумму и кнопку оплаты. Если у банка есть готовая ссылка/deeplink, вставь ее в `PAYMENT_URL_TEMPLATE`. Если ссылки нет, бот покажет реквизиты СБП, а покупатель нажмет `Я оплатил` и отправит чек.

## Важное про личный счет

Для личного счета по номеру телефона бот не может сам надежно увидеть входящий перевод без банковского API или уведомлений банка. Поэтому сейчас сделан рабочий ручной контур: клиент оплатил, нажал `Я оплатил`, прислал чек, админ подтверждает командой:

```text
/paid FS-1234567
```

После подтверждения бот пишет клиенту, что оплата принята. Если настроить SMTP, он отправит письма клиенту и магазину.

## Если сайт отдельно, а бот/API на Render

На Render подними этот проект как Web Service со стартовой командой:

```bash
npm start
```

В настройках Render добавь переменные из `bot/.env.example`. После деплоя скопируй URL Render и вставь его в `app.js`:

```js
const API_BASE_URL = "https://your-render-service.onrender.com";
```

После этого статический сайт можно выкладывать на обычный хостинг, а заказы будут улетать на Render, где их увидит бот.
